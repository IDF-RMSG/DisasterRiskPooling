# Countries data

## Files

| File | Description |
|---|---|
| `emdat_country_losses.csv` | **Active data file used by the app.** Country-level disaster loss data formatted for the Loss Simulator. Generated by `process_emdat.py`. |
| `process_emdat.py` | Python script that converts a raw EM-DAT download into `emdat_country_losses.csv`. |
| `public_emdat_download_*` | Raw EM-DAT export file(s). Not read by the app directly. |
| `emdat_country_occ.csv` | Previous version of the country losses data (superseded). |

## How to update `emdat_country_losses.csv`

### 1. Download fresh data from EM-DAT

1. Go to [https://www.emdat.be](https://www.emdat.be) and log in.
2. Under **Data > Public**, query for all countries and all relevant disaster types (Drought, Earthquake, Flood, Storm).
3. Export as **CSV**, saving the file into this folder (`LossSimulator/data/Countries/`).

### 2. Update the input filename in `process_emdat.py`

Open `process_emdat.py` and update line 5 to match the new filename:

```python
INPUT_FILE = SCRIPT_DIR / "public_emdat_download_YYYY-MM-DD_forLossSimulator.csv"
```

### 3. Run the script

From any directory:

```
python LossSimulator/data/Countries/process_emdat.py
```

Or from within this folder:

```
python process_emdat.py
```

The script will overwrite `emdat_country_losses.csv`.


## Notes

- **Damage units:** EM-DAT exports damage in thousands USD (`'000 US$`). The app's `global.R` multiplies damage values by 1,000 on load, so no manual conversion is needed.
- **Storm → Cyclone:** The script automatically renames `Storm` to `Cyclone` to match the app's peril naming convention.
- **Date range:** The app filters data to years 2002–2024 (set by `start_year` / `end_year` in `global.R`). Rows outside that range are loaded but ignored.
